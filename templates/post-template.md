# NN · Title

> **TL;DR.** Two sentences. The first names the post's claim; the second
> names what the post will deliver.
>
> **Reading time:** ~N minutes.
>
> **After reading this you will be able to:**
> - One concrete capability the reader gains. Verb first.
> - A second concrete capability.
> - A third concrete capability.

## 1. The motivation

Open with the symptom or question this post resolves. Concrete, not abstract.

## 2. The mechanism

Explain the underlying mechanism in plain language. One diagram if it helps.

## 3. The technique

Show the technique. Code or pseudo-code is fine. Cite the source if it has one.

## 4. The numbers

Whenever possible, attach a measurement: tokens, latency, percent-improvement,
benchmark score. No invented numbers; cite each one.

## 5. The variations

Two or three variants of the technique, each with the situation where it
shines.

## 6. When *not* to use it

A short, honest section on the edge of applicability.
